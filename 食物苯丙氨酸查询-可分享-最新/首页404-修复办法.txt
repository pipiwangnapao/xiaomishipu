========================================
  打开站点显示 "Page not found" 的修复办法
========================================

原因：Netlify 在「发布目录」里找不到 index.html。
多数情况是：Git 仓库里 index.html 在子文件夹里，而 Netlify 发布的是仓库根目录。

--------------------------------------
办法一：改 Netlify 的「Base directory」（推荐，最快）
--------------------------------------
1. 登录 https://app.netlify.com，点进你这个站点（moonlit-rolypoly-1cfa41）。
2. 左侧点「Site configuration」→「Build & deploy」→「Build settings」。
3. 找到「Base directory」：
   • 若你现在是空的，改成你的子文件夹名（和 GitHub 上一致），例如：
     食物苯丙氨酸查询-可分享
   • 若你确认 GitHub 仓库根目录就有 index.html（没有子文件夹），则 Base directory 保持空。
4. 「Publish directory」保持为空（或 .）。
5. 保存后，到「Deploys」→「Trigger deploy」→「Deploy site」，等部署完成再打开站点链接。

--------------------------------------
办法二：把文件移到仓库根目录（一劳永逸）
--------------------------------------
1. 打开你的 GitHub 仓库页面，看根目录下有什么。
2. 若根目录下是一个文件夹（例如「食物苯丙氨酸查询-可分享」），点进去才看到 index.html 和 netlify，说明需要把「可分享」里的内容提到根目录。
3. 在电脑上操作：
   • 打开「食物苯丙氨酸查询-可分享」文件夹，选中里面的全部内容：
     index.html、netlify.toml、netlify 文件夹、以及所有 .txt 文件。
   • 把这些直接上传到 GitHub 仓库的「根目录」（不要放进任何子文件夹），覆盖或替换原有内容，保证仓库根目录下就有 index.html 和 netlify 文件夹。
4. Netlify 里「Base directory」改为空（留空），保存。
5. 「Deploys」→「Trigger deploy」→「Deploy site」，再访问站点。

--------------------------------------
如何确认是否修好
--------------------------------------
用浏览器打开：https://你的站.netlify.app/
应看到「食物苯丙氨酸查询」的搜索页面，而不是 "Page not found"。
